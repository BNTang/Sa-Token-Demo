# 面向对象编程与面向过程编程的区别

## 核心区别

```
┌─────────────────────────────────────────────────────────────┐
│              OOP (面向对象) vs POP (面向过程)                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   面向过程 (POP):                                            │
│   ├── 关注"做什么" (过程/步骤)                              │
│   ├── 程序 = 数据 + 函数                                    │
│   ├── 函数操作数据                                          │
│   └── 代表语言: C                                           │
│                                                             │
│   面向对象 (OOP):                                            │
│   ├── 关注"谁来做" (对象)                                   │
│   ├── 程序 = 对象的集合                                     │
│   ├── 对象封装数据和行为                                    │
│   └── 代表语言: Java, C++, Python                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 代码对比

```c
// 面向过程 (C 语言风格)
struct User {
    char name[50];
    int age;
};

void printUser(struct User* user) {
    printf("Name: %s, Age: %d\n", user->name, user->age);
}

void setAge(struct User* user, int age) {
    user->age = age;  // 数据暴露，可以随意修改
}

int main() {
    struct User user = {"Tom", 18};
    setAge(&user, 20);
    printUser(&user);
}
```

```java
// 面向对象 (Java)
public class User {
    private String name;  // 封装
    private int age;
    
    public User(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    public void setAge(int age) {
        if (age > 0 && age < 150) {  // 可以加验证
            this.age = age;
        }
    }
    
    public void print() {
        System.out.println("Name: " + name + ", Age: " + age);
    }
}

// 使用
User user = new User("Tom", 18);
user.setAge(20);
user.print();
```

## OOP 三大特性

```
┌─────────────────────────────────────────────────────────────┐
│                    OOP 三大特性                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   1. 封装 (Encapsulation)                                   │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  • 隐藏内部实现，只暴露接口                          │  │
│   │  • private 修饰属性，public 方法访问                 │  │
│   │  • 可以加验证逻辑，保护数据                          │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   2. 继承 (Inheritance)                                     │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  • 子类复用父类的属性和方法                          │  │
│   │  • 代码复用，减少重复                                │  │
│   │  • 体现 "is-a" 关系                                  │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   3. 多态 (Polymorphism)                                    │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  • 同一行为多种表现形式                              │  │
│   │  • 父类引用指向子类对象                              │  │
│   │  • 提高扩展性和灵活性                                │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 详细对比

```
┌─────────────────────────────────────────────────────────────┐
│                    详细对比                                  │
├──────────────────┬──────────────────┬───────────────────────┤
│   维度           │   面向过程       │   面向对象            │
├──────────────────┼──────────────────┼───────────────────────┤
│   基本单元       │ 函数             │ 类/对象               │
│   关注点         │ 过程、步骤       │ 对象、职责            │
│   数据与行为     │ 分离             │ 封装在一起            │
│   代码复用       │ 函数复用         │ 继承、组合            │
│   扩展性         │ 较差             │ 较好                  │
│   可维护性       │ 复杂后难维护     │ 更易维护              │
│   执行效率       │ 高（直接调用）   │ 略低（方法调用开销）  │
│   适用场景       │ 简单任务、底层   │ 复杂业务、大型系统    │
└──────────────────┴──────────────────┴───────────────────────┘
```

## 面试回答

### 30秒版本

> **面向过程**关注"做什么"（步骤），程序是函数的组合，数据和行为分离。**面向对象**关注"谁来做"（对象），程序是对象的集合，数据和行为封装在一起。OOP 三大特性：**封装**（隐藏实现）、**继承**（代码复用）、**多态**（扩展性）。面向对象更适合复杂系统，面向过程执行效率高适合简单任务。

### 1分钟版本

> **核心区别**：
> - 面向过程：关注步骤，函数操作数据
> - 面向对象：关注对象，对象封装数据和行为
>
> **例子**：把大象装冰箱
> - 过程：1.开门 2.放入 3.关门
> - 对象：冰箱.开门()；冰箱.存入(大象)；冰箱.关门()
>
> **OOP 三大特性**：
> - 封装：隐藏实现，保护数据
> - 继承：代码复用
> - 多态：扩展性
>
> **适用场景**：
> - 面向过程：简单任务、性能敏感、底层开发
> - 面向对象：复杂业务、大型系统、需要扩展
>
> **Java 选择**：
> - Java 是纯面向对象语言
> - 一切皆对象（基本类型除外）
